> 版本：v1.0 | 创建日期：2026-06-08 | 状态：待确认

## 一、产品概述

### 1.1 产品定位

DeepSeek Notes 是一款**本地 AI 知识管理工具**，将 DeepSeek 大模型的对话能力与个人笔记管理无缝融合。用户可以在 AI 对话中即时捕获灵感，一键归档为结构化笔记，并通过笔记上下文驱动更深度的 AI 协作。

### 1.2 目标用户

- 知识工作者（研究员、产品经理、程序员）
- 学生（整理课程笔记、辅助学习）
- 内容创作者（文章素材管理、头脑风暴）

### 1.3 核心价值主张

> **「对话即笔记，笔记即上下文」**

1. 对话过程中随手存笔记，无需切换工具
2. 笔记作为上下文回流进 AI 对话，形成知识闭环
3. 所有数据本地存储，隐私安全可控

## 三、Tab 1：AI 笔记

### 3.1 布局概览

```
┌──────────────┬────────────────────────────┬─────────────┐
│  历史对话栏   │        AI 对话区            │   书签栏    │
│  (固定宽度)   │      (flex 自适应)          │ (固定宽度)  │
│              │                            │             │
│ - 对话列表   │  消息气泡区域（可滚动）       │ 本次已存笔记 │
│ - 搜索框     │                            │             │
│ - 新建对话   │  ─────────────────────     │             │
│              │  输入框（固定底部）          │             │
└──────────────┴────────────────────────────┴─────────────┘
```

### 3.2 左侧：历史对话栏

| 功能 | 描述 |
|------|------|
| 对话列表 | 展示标题 + 最后更新时间，按时间倒序 |
| 对话搜索 | 顶部搜索框，实时过滤 |
| 新建对话 | 顶部 `+` 按钮，清空中间区域 |
| 右键菜单 | 重命名、删除、导出为 Markdown |
| 自动命名 | 新对话首条消息后，AI 自动生成对话标题 |
| 持久化 | 所有对话存储于 SQLite，应用重启后保留 |

### 3.3 中间：AI 对话区

#### 3.3.1 消息展示

- 用户消息：右对齐气泡，浅色背景
- AI 消息：左对齐气泡，默认白色/暗色背景
- 支持 **Markdown 渲染**（含代码高亮、表格、数学公式）
- 支持**流式输出**（打字机效果）
- 消息操作悬浮按钮：
  - 📋 复制消息内容
  - 📌 一键存入笔记本（触发存笔记流程）
  - 🔄 重新生成（仅 AI 消息）

#### 3.3.2 输入框

```
┌──────────────────────────────────────────────────┐
│  📎 引用笔记    🌐 联网搜索 [●]    📄 附件         │  ← 工具栏
├──────────────────────────────────────────────────┤
│  在此输入消息...                                   │
│                                              发送  │
└──────────────────────────────────────────────────┘
```

| 功能 | 描述 |
|------|------|
| 文本输入 | 支持 Shift+Enter 换行，Enter 发送 |
| 附件上传 | 支持图片（PNG/JPG/WebP）、文档（PDF/DOCX/TXT/MD） |
| 联网搜索 | Toggle 开关，开启后 AI 回答时附带网络搜索结果 |
| 笔记引用 | 弹出笔记选择器，选中笔记以 `[[笔记标题]]` 格式插入上下文 |
| 字符计数 | 实时显示预计 Token 数 |

#### 3.3.3 一键存入笔记本流程

1. 用户点击消息上的 📌 按钮
2. 弹出"存入笔记"面板：
   - 自动填充：AI 生成标题建议（可修改）
   - 选择分类（下拉选择已有分类或新建）
   - 添加标签（多选）
   - 来源标注自动填充（来自对话 X，时间戳）
3. 确认后笔记写入数据库，书签栏实时刷新

### 3.4 右侧：书签栏

- 展示**当前会话**中已存入笔记本的所有笔记
- 每个书签卡片显示：笔记标题 + 存入时间
- 点击跳转到笔记本对应笔记
- 支持悬停预览（Tooltip 显示前 100 字）
- 会话切换时书签栏联动刷新

## 四、Tab 2：笔记本

### 4.1 布局概览

```
┌────────────────┬───────────────────────────────┬─────────────┐
│  分类/标签栏   │         笔记展示区              │  快捷方式栏  │
│               │  [搜索框]  [列表|卡片]  [━━○━━] │             │
│ 📁 所有笔记   │                               │  常用笔记   │
│ 📁 分类A      │  [笔记卡片] [笔记卡片]          │  快速访问   │
│  └ 📁 子分类  │  [笔记卡片] [笔记卡片]          │             │
│               │                               │             │
│ 标签           │                               │             │
│ 🏷 标签A      │                               │             │
└────────────────┴───────────────────────────────┴─────────────┘
```

### 4.2 左侧：分类与标签栏

#### 分类管理

- 树形目录结构，支持多级嵌套（最多 3 级）
- 内置节点：「所有笔记」「今日新增」「最近编辑」「未分类」
- 操作：新建分类、重命名、删除（含子分类的二次确认）、**拖拽排序**
- 点击分类自动过滤中间展示区

#### 标签管理

- 标签列表（支持颜色标记）
- 操作：新建标签（含颜色选择）、删除、**拖拽排序**
- 点击标签自动过滤中间展示区

### 4.3 中间：笔记展示区

#### 顶部工具栏

| 控件 | 功能 |
|------|------|
| 搜索框 | 关键词搜索（SQLite FTS5 全文检索）；语义搜索预留接口（v2） |
| 视图切换 | 列表视图 / 卡片视图 |
| 卡片大小 | 滑轨调节卡片宽度（S/M/L/XL，影响每行卡片数量） |
| 排序 | 按时间/标题/字数排序 |
| 新建笔记 | `+` 按钮，打开空白编辑弹窗 |

#### 笔记卡片

**卡片视图：**
```
┌─────────────────────────────┐
│ 🔴 [分类] [标签A] [标签B]   │  ← 颜色标记 + 分类标签
│                             │
│ 笔记标题                    │
│ 正文预览（前 3 行）...       │
│                             │
│ 2026-06-08  526字           │  ← 元数据
│ ✏️ 📌 💬 🗑️               │  ← 操作按钮
└─────────────────────────────┘
```

| 操作按钮 | 功能 |
|------|------|
| ✏️ 编辑 | 打开笔记编辑弹窗 |
| 📌 快捷方式 | 将此笔记添加到右侧快捷方式栏 |
| 💬 AI 对话 | 切换至 AI 笔记 Tab，以此笔记为上下文新建对话 |
| 🗑️ 删除 | 二次确认后删除 |

#### 笔记编辑弹窗

```
┌──────────────────────────────────────────────────┐
│  📄 笔记标题                        [主题▼] [✕]   │
├──────────────────────────────────────────────────┤
│ [B] [I] [~] [H1][H2][H3] [```] [—] [🔗] [🖼] [☑]│  ← Markdown 工具栏
├──────────────────────────────────────────────────┤
│                                                  │
│     [编辑区]              [预览区]               │  ← 左右分栏（可切换模式）
│                                                  │
├──────────────────────────────────────────────────┤
│  字数: 526  Token≈312   [历史版本 ▼]  [保存]      │
└──────────────────────────────────────────────────┘
```

**Markdown 5 种主题：**

| 主题名 | 风格 |
|--------|------|
| GitHub | 标准 GitHub 风格，干净专业 |
| Dark | 深色背景，护眼夜间阅读 |
| Minimal | 极简排版，留白充足 |
| Academic | 衬线字体，学术论文风 |
| Terminal | 等宽字体，绿色终端风格 |

**版本历史：** 每次保存自动创建快照，最多保留 30 个版本

### 4.4 右侧：快捷方式栏

- 用户手动添加的笔记快速入口（最多 20 条）
- 支持拖拽排序
- 点击直接打开笔记编辑弹窗
- 右键移除快捷方式

### 5.2 API 配置模块

```
┌─────────────────────────────────────────────┐
│  模型提供商：[DeepSeek ▼]                   │
│  API Key：  [sk-•••••••••••••••]  [测试连接]  │
│  模型选择：  [deepseek-v4-flash ▼]          │
│  API 地址：  https://api.deepseek.com       │  ← 可修改（兼容 OpenAI 格式）
│                                             │
│  + 添加其他模型提供商                       │  ← 支持扩展 OpenAI / 自定义
└─────────────────────────────────────────────┘
```

**支持的模型：**

| 提供商 | 模型 | 说明 |
|--------|------|------|
| DeepSeek（默认） | deepseek-v4-flash | 通用对话，速度快，上下文 1M |
| DeepSeek | deepseek-v4-pro | 深度推理，适合复杂分析，上下文 1M |
| 自定义 | 用户填写 | 兼容 OpenAI 格式的任意服务 |

**安全：** API Key 通过 electron-store 加密存储，不以明文写入数据库

## 六、银月补充需求

以下是在 Thom 原始需求基础上建议补充的功能点：

### 6.1 对话增强
- **系统提示词（System Prompt）**：用户可为每个对话自定义 AI 角色设定
- **上下文长度控制**：配置携带历史消息的轮数（防止 Token 超限）
- **对话模板**：预设常用提示词（摘要笔记、续写、翻译、解释代码等）
- **消息编辑**：支持编辑已发送的消息后重新触发 AI 回复

### 6.2 笔记增强
- **笔记标题自动生成**：存入时 AI 提供标题建议（可关闭）
- **颜色标记**：笔记卡片支持 6 种颜色标记，方便视觉区分
- **笔记版本历史**：编辑弹窗内可查看历史版本并对比/还原
- **关键词高亮**：搜索结果中高亮匹配关键词

### 6.3 体验优化
- **键盘快捷键**：
  - `Ctrl/⌘ + N`：新建对话
  - `Ctrl/⌘ + Shift + F`：全局搜索
  - `Ctrl/⌘ + ,`：打开设置
  - `Escape`：关闭弹窗
- **全局搜索**：`Ctrl+Shift+F` 跨对话+笔记搜索
- **自动保存**：笔记编辑 2 秒无操作自动保存草稿
- **离线提示**：联网搜索不可用时友好提示

### 6.4 数据安全
- **自动备份提醒**：每 7 天提示用户导出备份
- **应用内数据统计**：清晰展示存储占用大小
