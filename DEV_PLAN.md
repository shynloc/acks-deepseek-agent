## 技术债与 v2 规划

### 暂缓到 v2 的功能

| 功能 | 原因 |
|------|------|
| 语义搜索（Embedding） | 需要 Embedding API，成本和复杂度较高 |
| 多窗口/分屏模式 | 架构改动较大 |
| 插件系统 | 过于复杂 |
| 云同步 | 需要后端服务 |
| iOS/Android 版本 | 跨平台重构 |

### 已知技术风险

| 风险 | 影响 | 缓解方案 |
|------|------|----------|
| Electron 包体积大（~150MB） | 用户安装体验 | 考虑 v2 迁移到 Tauri |
| better-sqlite3 需要 native 编译 | 跨平台打包 | electron-rebuild 处理 |
| DeepSeek API 响应慢时 UX 差 | 用户体验 | 流式输出 + 取消按钮 |
| 大量附件占用本地存储 | 磁盘空间 | 附件大小限制 + 清理工具 |

## 开发环境准备

```bash
# 前置要求
node >= 20
npm >= 9

# 初始化项目
git clone <repo>
cd deepseek-notes
npm install

# 开发
npm run dev

# 打包
npm run build
npm run package   # 生成安装包
```
